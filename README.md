#加贸云

### 目录结构

    common 中仅能放置全局通用的一些代码，若代码与某个业务模块强耦合则不允许放入本包下
    
    config 全部项目中的配置，模块内部不允许再进行配置
    
    handler 处理类，模块内部不允许定义全局处理，全局处理必须放在本目录下
    
    其它的同级包均为业务模块分类，具体业务模块内部所需要的POJO，枚举应都要定义到自己的模块包下

1. common(通用代码)
    
    > enums(全局通用枚举)
    
    > exception(全局自定义异常)
    
    > model(全局通用POJO)
    
    > util(全局通用工具类)
    
2. config(全局配置)

3. handler(全局处理)

4. portal(门户模块)

    > JWT 身份认证
